How should events data be managed?

This determines how you add/update events and how the page stays current

A

Static JSON files (Recommended)

Events defined in content/events/ as JSON files, just like courses. You manually add events and include the lu.ma URL for each. Simple, consistent with existing patterns, no API keys needed.

Pros

  • Matches existing content pattern
  • No external dependencies
  • Full control over display
  • Works at build time (fast)

Cons

  • Manual updates when adding events
  • Won't auto-sync with lu.ma
B

Lu.ma API Integration

Fetch events directly from lu.ma's API at build time or runtime. Events auto-sync when you publish on lu.ma. Requires API access.

Pros

  • Auto-syncs with lu.ma
  • No manual content updates
  • Always current

Cons

  • Depends on lu.ma API availability
  • May need API key
  • Less control over data shape
  • New external dependency
C

Hybrid: Static JSON + Lu.ma link

Static JSON for event metadata (title, date, location, tags), but each event links out to its lu.ma page for registration. Best of both worlds.

Pros

  • Full design control
  • Lu.ma handles registration
  • Consistent with codebase
  • Easy to add events

Cons

  • Still manual updates

Step 5 • Second opinion

PRD review with codex

Independent adversarial review — a different model, different training, zero context on my chat history. Finds things Claude won't.

  • ✓ "You mention 'approved' but no auth for approvers."
  • ✓ "Rate limit is per-cookie — trivial to bypass."
  • ✓ "What happens when a talk is rejected after a vote?"

Value of adversarial review

Claude is optimized to help you finish. Codex, invoked in "challenge" mode, is optimized to poke holes.

Result: 3 real gaps closed before a single line of code.

Speaker Notes

08:30 – 10:00 (1.5 min)
  • codex review skill wraps OpenAI's codex CLI. Think of it as a 200-IQ grumpy senior who hasn't seen your conversation.
  • Always review PRD, plan, and diff through a different model. Different blind spots.

Step 7 • Quality gate

Code review + E2E test

Three-lens review before any merge:

  • ✓ Claude self-review (simplify skill)
  • ✓ codex review (adversarial)
  • ✓ Human eye on the diff

Plus Playwright E2E: submit → approve → vote → verify.

E2E: the good & the trap

# good: real end-to-end flow await page.goto('/submit'); await page.fill('name', speaker); await page.click('submit'); # trap: every CI run created # real records in prod KV "E2E Speaker 1776673201567" "E2E Speaker 1776673201912" ...forever.

Speaker Notes

12:00 – 13:30 (1.5 min)

Step 9 • Root cause

The env config trap

Vercel KV was deprecated. My KV store migrated to Upstash — which provisions different env var names.

  • ✓ @vercel/kv reads KV_REST_API_URL
  • ✓ Upstash provides UPSTASH_REDIS_REST_URL
  • ✓ First kv.incr() throws
  • ✓ No try/catch → 500 HTML page
  • ✓ Client default case → wrong error text

The fix (PR #66)

// src/lib/kv.ts if (!process.env.KV_REST_API_URL && process.env.UPSTASH_REDIS_REST_URL) { process.env.KV_REST_API_URL = process.env.UPSTASH_REDIS_REST_URL; } export { kv } from '@vercel/kv';

Speaker Notes

15:00 – 17:00 (2 min)
  • The environment changed under me. The package didn't fail at install — it failed at first call.
  • Fix: alias the env vars at module load before the lazy KV proxy reads them.
  • Also wrapped every voting route in try/catch so real errors hit Vercel logs instead of hiding in HTML 500s.
  • Lesson: always have a path from "prod broke" to "I can see the stack trace in 30 seconds."
Recap

5 lessons from this build

1

Start with a scope-killer — Hermes / office-hours before any coder.

2

Match the tool to the stage — Claude Code doesn't write PRDs well; superpowers does.

3

Always get a second model's eyes — codex catches what Claude rationalizes.

4

"Tests pass" ≠ "prod works" — env, secrets, and integrations live outside your repo.

5

Make prod errors observable — try/catch every KV/API call; log with a tag.

6

E2E tests need teardown — or they pollute real data. If you can't clean up, delete the test.

Speaker Notes

17:00 – 19:00 (2 min)
  • The lessons aren't about AI tools — they're about when each AI tool is the right hammer.
  • #4 and #5 are the ones that saved me most time after the prod incident.

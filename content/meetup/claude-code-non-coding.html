What Claude Code actually does

It is a CLI agent with full access to your project directory.

  • ✓ Reads and writes any file
  • ✓ Runs shell commands
  • ✓ Follows complex, multi-step prompts
  • ✓ Remembers context across a session
  • ✓ Executes reusable Skills

Example Session

$ claude
# You are now in a conversation
# with an agent that can:

• Read your project files
• Write HTML, Markdown, JSON
• Run npm, git, ffmpeg
• Follow skill instructions
• Iterate based on feedback

Speaker Notes

05:00 – 07:00 (2 min)
  • Key point: Claude Code is not an IDE plugin. It is a terminal-based agent.
  • It has the same capabilities whether you are writing TypeScript or writing a blog post — it reads files, writes files, and runs commands.
  • The “Skills” system lets you teach it reusable workflows, which is the key to scaling non-coding work.

Skills

What is a Skill?

A Markdown file that teaches Claude Code a reusable workflow. Lives in your project under .claude/skills/.

  • ✓ Plain Markdown — no code needed
  • ✓ Invoked with /skill-name
  • ✓ Can reference files, templates, styles
  • ✓ Shareable across your team
  • ✓ Version-controlled with your project

Skill File Structure

# .claude/skills/create-slide/

SKILL.md # Instructions
assets/
  base-theme.css # CSS theme
references/
  slide-patterns.md # HTML patterns

Speaker Notes

18:00 – 20:00 (2 min)
  • A Skill is just a Markdown file with instructions. Think of it as a reusable prompt with supporting files.
  • The create-slide skill has: a SKILL.md with step-by-step instructions, a CSS theme file, and an HTML patterns reference.
  • When I type /create-slide, Claude Code reads the skill file, follows the instructions, reads the supporting files, and produces the output.
  • Key benefit: consistency. Every slide deck looks the same, follows the same patterns, and has the same quality bar.
Skills

Inside /create-slide

The skill tells Claude Code exactly what to do, step by step.

  • 1 Gather requirements from user
  • 2 Read the base CSS theme
  • 3 Read the HTML slide patterns
  • 4 Compose slides using patterns
  • 5 Add speaker notes + timing
  • 6 Output two HTML files

From SKILL.md

## Workflow

1. Gather requirements: topic,
   audience, duration
2. Read assets/base-theme.css
3. Read references/slide-patterns.md
4. Compose slides using patterns
5. Add speaker notes with timing
6. Verify: required types present,
   timing sums to target duration

Best Practices

Best Practices for Non-Coding Work

🎯

Be Specific

Give context: audience, format, length, tone. The more detail, the better the output.

📁

Use Project Context

Work inside your project directory. Claude Code reads existing files to match your patterns.

🔄

Iterate, Don’t Restart

Give feedback in the same session. “Make it shorter.” “Add a section on X.”

Speaker Notes

26:00 – 27:30 (1.5 min)
  • Be specific: “Write a blog post” is vague. “Write a 1000-word blog post about X for non-technical audience in conversational tone” is much better.
  • Use project context: Always run Claude Code from your project root. It will read existing files and match conventions.
  • Iterate: The first output is a draft. Refine it in conversation. This is faster than starting over.
Best Practices

When to create a Skill

Not every task needs a skill. Here is my rule of thumb.

  • ✓ You do it more than 3 times
  • ✓ Output quality must be consistent
  • ✓ It references specific files or templates
  • ✓ Others on your team will do it too

My Skills at 02Ship

Claude Code is a work tool, not just a coding tool. Any text-based task is fair game.

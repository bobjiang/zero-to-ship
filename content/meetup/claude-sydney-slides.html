Section 1 • What is Claude

What is Claude?

An AI assistant by Anthropic — built to be helpful, harmless, and honest.

💬

Conversational

Chat naturally — describe what you need in plain language, not commands.

🧠

Reasoning

Extended thinking for deep analysis, planning, and multi-step problems.

🌐

Multi-modal

Understands text, images, PDFs, code, and structured data.

Speaker Notes

3:00 – 6:00 (3 min)
  • Claude is Anthropic's flagship AI assistant. Think of it as a knowledgeable colleague you can have a conversation with.
  • Three key properties: (1) conversation-first interface, (2) built-in reasoning (extended thinking scales to problem difficulty), (3) multimodal — upload screenshots, documents, data.
  • Stress that you don't need to "code" to use Claude. Describe what you want in everyday language.
Section 1 • Models

Three Models, One Family

Choose the right model for speed, depth, or balance.

Haiku

Fastest • Lightest

Quick answers, summaries, classification, simple extraction. Lowest token cost.

Sonnet

Daily Driver

Coding, writing, analysis, multi-step workflows. Best balance of speed and quality.

Opus

Deepest Reasoning

Complex research, sustained thinking, accuracy-critical tasks. Most capable model.

Speaker Notes

6:00 – 8:00 (2 min)
  • Haiku, Sonnet, Opus — think of them as lightweight, mid-weight, heavyweight.
  • Free plan: Haiku + Sonnet. Pro/Max: adds Opus with higher limits.
  • Both Sonnet 4.6 and Opus 4.6 feature adaptive extended thinking — they scale reasoning depth to problem complexity automatically.
  • Rule of thumb: start with Sonnet. Upgrade to Opus only when Sonnet's answer isn't deep enough. Use Haiku for bulk/simple tasks.
Section 2 • Chat

Claude Chat

Quick, conversational interactions. Your go-to for questions, brainstorming, and iterative problem-solving.

  • ✓ Double-tap Option (Mac) for instant access
  • ✓ Share screenshots and windows
  • ✓ Voice dictation for hands-free input
  • ✓ Projects to organize related conversations
  • ✓ Artifacts for code, docs, and visuals

Speaker Notes

8:00 – 12:00 (4 min)
  • Chat is the default tab when you open Claude. It's the most familiar interface — just like texting.
  • Show the desktop app: double-tap Option to summon Claude as an overlay. Great for quick lookups while working.
  • Projects: group related conversations together. You can attach files, set custom instructions, and maintain context across chats.
  • Artifacts: when Claude generates code, a document, or a visualization, it appears as an interactive artifact you can edit, copy, or share.

Section 3 • Cowork

Claude Cowork

Complex, sustained work — research, analysis, and document creation. Claude reads your files, browses the web, and produces deliverables.

  • ✓ Access local folders — read and create files
  • ✓ Web browsing for live research
  • ✓ Scheduled tasks for recurring work
  • ✓ Plugin ecosystem for extended tools
  • ✓ Protected sandbox environment

Best For

Research briefs from multiple sources
Contract and document review
Cross-source analysis & reports
Comprehensive document creation

Speaker Notes

16:00 – 20:00 (4 min)
  • Cowork is the newest mode — think of it as giving Claude a desk, a browser, and a file cabinet.
  • Key difference from Chat: Cowork can access local files and browse the web. It's designed for tasks that take 10-30+ minutes of focused work.
  • Scheduled tasks: set Claude to run weekly competitor analysis, daily report summaries, etc.
  • Sandbox: Claude works in a safe environment — it can create files but can't break your system.
  • Requires Pro, Max, Team, or Enterprise plan.

Section 4 • Code

Claude Code

A full development environment in your terminal. Claude reads your codebase, writes code, runs tests, and commits — with your approval.

  • ✓ Direct codebase access with visual diffs
  • ✓ Built-in terminal for running commands
  • ✓ Git integration and version tracking
  • ✓ Three modes: Ask, Code, Plan
  • ✓ Works locally or on GitHub remote

Terminal Session

$ claude # Claude Code reads your project > Analyzing codebase... 47 files indexed You: Add dark mode to the settings page > I'll add dark mode support. Here's my plan: 1. Add theme toggle component 2. Update CSS variables 3. Persist preference in localStorage [Apply changes? y/n]

Speaker Notes

24:00 – 28:00 (4 min)
  • Claude Code is the power-user tool — a CLI agent that lives in your terminal.
  • It indexes your entire codebase and understands project structure, dependencies, and patterns.
  • Three interaction modes: Ask (requires approval for each change), Code (auto-applies safe changes), Plan (reviews strategy before implementation).
  • You remain in control: Claude shows diffs before applying, and you approve or reject each change.
  • Also available in the Claude desktop app under the Code tab (rolling out). Requires Pro+ plan.

Speaker Notes

34:00 – 38:00 (4 min)
  • Walk through the table row by row. The key insight: each mode is optimized for a different kind of work.
  • Chat = quick, conversational, no file access needed.
  • Cowork = sustained, research-heavy, needs file/web access, produces documents.
  • Code = software development, needs codebase access, runs commands, creates diffs.
  • Overlap exists — and that's fine. The table shows where each mode excels, not the only place it works.
Section 5 • Decision Guide

Quick Decision Guide

Ask yourself one question: what am I producing?

💬

An Answer

Questions, brainstorms, drafts, explanations, one-off help.

Use Chat
📄

A Document

Research briefs, reports, analysis, file-based deliverables.

Use Cowork
💻

Working Software

Features, bug fixes, refactors, tests, deployable code changes.

Use Code

Speaker Notes

38:00 – 40:00 (2 min)
  • Simplest mental model: what are you producing? An answer, a document, or working software?
  • This isn't a rigid rule — it's a starting point. You'll develop intuition over time.
  • Pro tip: you can start in Chat to brainstorm, then switch to Cowork or Code when you're ready to execute.
Section 5 • Models

Choosing the Right Model

Start with Sonnet. Escalate to Opus when depth matters.

Haiku

Categorize, extract, summarize, classify. Bulk tasks with simple logic.

Free plan included

Sonnet

Code, write, analyze, debug. Best speed-to-quality ratio. Your daily default.

Free plan included

Opus

Deep research, complex reasoning, accuracy-critical tasks. When Sonnet isn't enough.

Pro / Max plan

Speaker Notes

40:00 – 42:00 (2 min)
  • Model choice is separate from the Chat/Cowork/Code decision. You can use any model in any mode.
  • Practical advice: default to Sonnet for everything. Switch to Opus if Sonnet's answer feels shallow or wrong. Use Haiku for high-volume, simple tasks (e.g., processing 100 support tickets).
  • Both Sonnet and Opus have adaptive extended thinking — they automatically spend more "thinking time" on harder problems.
  • Each new model generation shifts which tasks fit where, so re-evaluate when new versions drop.

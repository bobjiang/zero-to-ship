Technique 1 • 0:45
01
Technique 1

Simplify

When the response is too complex, uses jargon, or feels overwhelming.
"Can you simplify this? Explain it like I've never coded before."
"Explain this like I'm five."
Before

"The HTTP request triggers DNS resolution, which maps the domain to an IP address. The server processes the request via server-side rendering..."

After

"Typing a web address is like mailing a letter to a specific house. Your browser finds the right house and brings back what's inside."

Technique 2 • 2:15
02
Technique 2

Expand

When the response is too brief, skips important details, or feels incomplete.
"Can you add more detail about step three?"
"What does this step actually involve?"

Key: Be specific about WHICH part needs expanding.

Before

"Step 3: Set up your workspace."

After

"Step 3: Set up your workspace. First, create a new folder called 'my-project'. Inside that folder, create three sub-folders: 'src' for your code, 'public' for images, and 'content' for your writing..."

Technique 3 • 3:30
03
Technique 3

Restructure

When the content is right but the format is wrong. Paragraphs when you need a checklist, or a list when you need a table.
"Format this as a numbered step-by-step guide."
"Turn this into a table with columns."
Before (Paragraph)

"A feature should validate the user input, then save it to the database, then show a confirmation message, and also log the event..."

After (Checklist)

☐ Validate user input
☐ Save to database
☐ Show confirmation message
☐ Log the event

Technique 4 • 4:45
04
Technique 4

Combine

When you have multiple options and want the best of each. You're the creative director, picking what works.
"Combine the structure from option A with the tone from option B."
"Take the best parts and create a hybrid version."
OPTION A: Great structure, too formal tone
OPTION B: Perfect casual tone, messy structure
↓ Combine ↓
HYBRID: Great structure + casual tone = Perfect
Technique 5 • 5:45
05
Technique 5

Constrain

When the response is too long, unfocused, or includes unnecessary complexity. Set boundaries on length, scope, or audience.
"Keep this under 100 words."
"Remove anything a beginner wouldn't need."
3 pages of detailed output
with advanced topics
↓ Constrain ↓
Focused,
essential info

Like cropping a photo to focus on what matters

Technique 6 • 7:00
06
Technique 6

Exemplify

When you understand the theory but need to see it in practice. Examples turn abstract ideas into something tangible.
"Can you show me a concrete example?"
"Show me what that looks like for /courses/claude-basics/lesson-1"
Theory

"Routing in Next.js maps file paths to URL paths using a folder-based convention..."

Example

src/app/courses/[series]/[lesson]/page.tsx
→ /courses/claude-basics/lesson-1

Technique 7 • 8:15
07
Technique 7

Alternative

When the first suggestion doesn't feel right or you want to compare options before committing.
"Give me three other ways to do this."
"What's a simpler alternative?"

Pairs well with Combine (Technique 4)!

Option A: Simple file-based approach
Option B: Database-driven approach
Option C: Hybrid with JSON files

Like shopping: try a few before you decide

Technique 8 • 9:30
08
Technique 8

Critique

When you want to validate quality or you're not sure if the response is good enough. Claude can honestly evaluate its own work.
"What are the weaknesses of this approach?"
"Review this for potential problems."
You
Critique your own answer. What did we miss?
C
Good catches needed:
• No error handling for failed saves
• Missing accessibility attributes
• No loading state for async operations

Your built-in quality checker

Chaining • 10:45

Chain Techniques Together

Simplify Exemplify
"Simplify this AND give me an example."
Alternative Combine
"Give me 3 alternatives, then combine the best parts."
Expand Restructure
"Add more detail and format it as a numbered list."
Keep it to 2-3 techniques per follow-up for clarity

1 Recipe 1

Build a Feature

The workhorse template for adding anything new to a project. Used more than any other while creating 02Ship.

Power move: "Before coding, please ask clarifying questions about data structure, edge cases, and integration points."

Template

I'm working on [project description].

I want to add [feature name] that allows users to [action].

Tech stack: [languages and frameworks]
Constraints: [any limits]

The feature should:
1. [requirement]
2. [requirement]
3. [requirement]

Before coding, please ask clarifying questions about data structure, edge cases, and integration points.

Filling in the template for a real 02Ship feature: a lesson progress tracker.

Filled-In Example

I'm working on a learning platform for non-programmers.

I want to add a lesson progress tracker that allows users to mark lessons as complete.

Constraints: No database yet -- use localStorage. Keep it simple.

1. Show a progress bar on each course page
2. Let users click to mark a lesson done
3. Remember progress when they come back

Before coding, please ask clarifying questions.

2 Recipe 2

Explain This Concept

Your learning accelerator. Use this whenever you encounter something you don't understand -- coding concepts, business terms, or confusing Claude output.

The magic: You're not just asking "what is X?" -- you're telling Claude exactly HOW you want to learn.

Template

I'm learning about [concept] for [context or purpose].

My current knowledge level: [beginner / intermediate / advanced]

Please explain [concept] by:
1. Using an analogy from everyday life
2. Breaking it into 3-5 key points
3. Showing a simple code example if applicable
4. Explaining one common mistake people make

3 Recipe 3

Review My Work

Your honest feedback partner. Most beginners never think to ask Claude for feedback -- but they should. Ask Claude to critique, not just create.

Key phrase: "Be honest but constructive -- I'm here to learn." This sets the perfect tone for useful feedback.

Template

I wrote [code / document / design] for [purpose].

Please review it and provide feedback on:
1. [specific aspect: code quality, clarity]
2. [specific aspect: performance, UX]
3. [specific aspect: edge cases, security]

My skill level: [beginner / intermediate]

Be honest but constructive -- I'm here to learn.

[Paste your work here]

4 Recipe 4

Brainstorm Solutions

The diverge-then-converge pattern baked into one prompt. Get a creative explosion AND structured evaluation in a single turn.

From Video 2: This is the three-step brainstorm pattern -- generate, evaluate, recommend -- all in one prompt.

Template

Problem: [describe in 1-2 sentences]

Context:
- Target users: [who]
- Constraints: [time, budget, tech limits]
- What I've tried: [previous attempts]

Please generate:
1. 10 creative solutions (quantity over quality)
2. Pick the top 3 -- pros and cons of each
3. Recommend one with your reasoning

5 Recipe 5

Create a Plan

Your secret weapon for getting started on any project. Instead of staring at a blank page wondering where to begin, hand the planning to Claude.

Real example: This exact template was used to plan 02Ship's 4-week launch. Claude even warned that video production would be the bottleneck.

Template

Goal: [what you want to achieve]
Current state: [where you are now]
Desired state: [where you want to be]
Timeline: [how long you have]

Please create a step-by-step plan that:
- Breaks the goal into weekly milestones
- Identifies potential roadblocks
- Suggests resources or learning needed
- Keeps the scope realistic for [timeline]

Section 1 • 1:15
Code Generation

Specificity Matters Most

Code prompts are where the 4 Cs shine brightest. Always specify four things:

1 Language / Framework
2 What the code should do
3 How it should be structured
4 Constraints (keep it simple, accessible)

Good Code Prompt

"Create a responsive navigation bar using React and Tailwind CSS. It should have: a logo on the left, 4 nav links in the center, and a dark mode toggle on the right. On mobile, collapse into a hamburger menu. Use TypeScript. Keep it under 60 lines."
Section 1 • 3:00

You Don't Need to Understand It to Use It

Your 02Ship Tech Stack Cheat Sheet

  • React -- builds the user interface
  • TypeScript -- adds safety to JavaScript
  • Tailwind CSS -- styles everything fast
  • Next.js -- powers the whole app

Section 2 • 4:00
Explanation Prompts

Claude as Your Personal Tutor

The key to great explanations is telling Claude your knowledge level. The same concept explained to a five-year-old sounds completely different from an explanation for a developer.

Tell Claude Your Level

If you're in this course, say "explain for a complete beginner"

Three Favorite Formats

"Explain like I'm 5"
"Give me an analogy from everyday life"
"Walk me through this step by step"

Section 3 • 6:30
Brainstorming

When You Need Ideas, Not Answers

Brainstorming is the opposite of code prompts. Code needs tight constraints. Brainstorming needs freedom. Ask for quantity first, then narrow down.

Quantity Over Quality

The magic usually happens in ideas 7 through 10.

Brainstorm Prompt

"I'm building a learning platform for non-programmers. I need creative ways to keep students engaged between lessons. Give me 15 ideas, ranging from conventional to wild and creative. Don't filter yourself -- quantity over quality."
Section 3 • 7:45

Diverge, Then Converge

🌈 15 creative ideas -- quantity over quality
▼
🔍 Top 3 with pros and cons
▼
✅ Final recommendation

Follow-up: "Pick the top 3 and explain the pros and cons of each. Then recommend one."

Section 4 • 9:30

Four Real Productivity Prompts

✍️ Writing

"Draft a professional but friendly email to my team announcing that our project deadline is moving to next Friday. Positive tone. Under 150 words."

🔍 Research

"Summarize this article in 5 bullet points. Focus on key arguments and data. Then tell me one thing the author might be wrong about."

📈 Analysis

"Do a SWOT analysis for an online learning platform that teaches non-programmers to build with AI tools. Be honest and critical."

📅 Planning

"Create a 4-week plan for launching a personal blog. I can spend 5 hours per week. Break into weekly milestones with specific tasks."
Section 5 • 11:30

Mix and Match Prompt Types

Code Explain
"Generate a React component for a newsletter signup form AND explain what each part of the code does in plain English."
Brainstorm Plan
"Brainstorm 10 feature ideas for my app, then pick the top 3, and create a one-week implementation plan for the best one."

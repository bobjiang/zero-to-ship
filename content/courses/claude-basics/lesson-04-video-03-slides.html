Folders Are Like Filing Cabinets

Code is organized in a folder structure. It looks complex at first, but follows logical patterns.

src/ = Source code (the main drawer)
components/ = Reusable building blocks (LEGO pieces)
app/ = Website pages (mirrors your URLs)

Section 2 - 3:00

Every Code File Has Three Parts

1. Imports Ingredient list -- what this file needs
import React from 'react';
import Button from './Button';
2. Logic The working instructions and what appears on screen
export default function Hero() {
  return (
    <h1>"Welcome"</h1>
  );
}
3. Export Label on the box -- makes it available to others
export default Hero;
Section 2 - 4:30

The Return Statement

The return statement is the most important part for you. It contains what actually appears on screen.

Everything inside return (...) is what the user sees.

return = What you see on screen

export default function HeroSection() { // Logic goes here (ignore for now) const title = "Welcome"; return ( ← THIS is what matters <section> <h1>{title}</h1> <p>Build your ideas</p> <Button>Get Started</Button> </section> ); }
Section 3 - 5:30

What You Can Safely Change

✓

Text in Quotes

Content displayed to users

<h1>"Welcome to 02Ship"</h1>
✓

Numbers

Size, spacing, timing, quantity

fontSize: 24 → fontSize: 36
✓

Colors

Hex codes (#3498db) or names

color: "#3498db" → "#e67e22"
✓

URLs and Paths

Image sources, links, file paths

src="/logo.png"
Section 4 - 7:30

What NOT to Change

✗

Brackets & Syntax

The grammar of code. Delete one and the file stops working.

{ } ( ) [ ] < /> ;
✗

Imports

Removing an import is like removing a recipe ingredient.

import React from 'react'
✗

Variable Names

Renaming breaks every reference. Like renaming your street.

const userName = "Bob"

Broke something? Press Cmd+Z / Ctrl+Z to undo

Section 5 - 9:00

Your Built-In Code Tutor

You
What does this function do?

export default function Hero() { return (<h1>Welcome</h1>); }
C
This creates a Hero component that displays a heading saying "Welcome." It's a reusable building block. To change the heading text, just edit the text between the <h1> tags.
You
How do I change the heading text in this component?
C
Find the text "Welcome" between the <h1> tags and replace it with whatever you'd like. For example: <h1>Hello from 02Ship</h1>

When in doubt, ask Claude

## Section 2: What Claude CAN Do (3:00 - 5:30)

### Three Core Capabilities

**SCRIPT:**
> "Claude has three superpowers that make it perfect for building products as a non-programmer:"

#### 1. Generate Code (30 seconds)

**Keypoints:**
- Creates complete, working code from your descriptions
- Handles multiple programming languages and frameworks
- Can build entire project structures, not just small snippets

**Example to mention:**
> "I can tell Claude, 'I want a landing page with a hero section and a sign-up form,' and it will generate all the HTML, CSS, and JavaScript needed. I don't need to know what those are."

#### 2. Explain Concepts (30 seconds)

**Keypoints:**
- Breaks down technical concepts in plain English
- Answers "why" questions, not just "how"
- Helps you understand what the code is doing without needing to become a programmer

**Example to mention:**
> "If Claude generates something and I don't understand it, I can ask, 'What does this part do?' and it will explain it in simple terms."

#### 3. Iterate on Ideas (1:00)

**Keypoints:**
- Takes feedback and refines the code
- Doesn't get frustrated with changes
- Helps you explore different approaches

**Example to mention:**
> "When I built 02Ship, my first attempt at the course page wasn't quite right. I told Claude, 'Can we add learning objectives and multiple videos per lesson?' and it updated everything. No starting from scratch."

**Visual suggestions:**
- Show split-screen: your prompt on one side, Claude's code output on the other
- Show a before/after example of iteration
- On-screen text highlighting the three capabilities

## Section 3: What Claude CAN'T Do (5:30 - 7:00)

### Important Limitations

**SCRIPT:**
> "Now, let's talk about what Claude can't do‚Äîbecause understanding the limits is just as important as knowing the strengths."

#### 1. Run Code for You (30 seconds)

**Keypoints:**
- Claude generates code, but you need an environment to run it
- This is actually good‚Äîit means you control what happens
- You'll need to set up basic tools (we cover this in Video 2)

**What to say:**
> "Claude can write the code, but it can't execute it on your computer. Think of it like a chef giving you a recipe‚Äîyou still need to cook it in your own kitchen."

#### 2. Deploy Apps (20 seconds)

**Keypoints:**
- Deployment means putting your app on the internet
- Claude can guide you through deployment, but you'll need to use services like Vercel or Netlify
- Good news: these services make deployment very easy (often one click)

#### 3. Access the Internet Without Tools (20 seconds)

**Keypoints:**
- Claude doesn't browse the web or pull live data by default
- It has knowledge up to its training date (January 2025)
- There are ways to give Claude internet access (advanced topic for later)

**Visual suggestions:**
- Use simple icons or graphics to show what's "out of bounds"
- Show a checklist: ‚úì Generate code, ‚úó Run code, ‚úó Deploy automatically
- Keep this section light and non-scary

## Section 4: The Mindset Shift (7:00 - 9:00)

### Claude as Collaborator, Not Magic Button

**SCRIPT:**
> "Here's the most important thing I want you to understand: Claude is not a magic button that builds your app in one click. It's a collaborator. And like any good collaboration, it works best when you bring clarity, direction, and feedback."

#### Think of Claude as a Smart Intern Who Can Code (45 seconds)

**Keypoints:**
- You provide the vision and direction
- Claude executes the technical work
- You review the results and give feedback
- Claude refines based on your input

**Analogy to use:**
> "Imagine you're a director and Claude is your film crew. You don't need to know how to operate the camera, but you do need to know what scene you want to shoot. Claude handles the technical execution; you handle the creative direction."

#### You Need to Know WHAT You Want (45 seconds)

**Keypoints:**
- Claude can't read your mind
- The clearer your description, the better the result
- You don't need to know HOW to build it (that's Claude's job)
- You DO need to know WHAT you want and WHO it's for

**Example to mention:**
> "When I started building 02Ship, I didn't know Next.js or TypeScript. But I did know: 'I want a learning platform with courses, videos, and a blog for non-programmers.' That was enough."

#### Iteration Is the Superpower (30 seconds)

**Keypoints:**
- Your first prompt won't be perfect‚Äîand that's completely normal
- Professional developers iterate constantly
- Each round of feedback makes the product better
- The goal isn't perfection on attempt one; it's progress through conversation

**What to emphasize:**
> "Every time I built a feature for 02Ship, it took 2-3 rounds of back-and-forth. First attempt: functional but basic. Second attempt: better design. Third attempt: exactly what I wanted. That's how building works‚Äîwith or without AI."

**Visual suggestions:**
- Show a simple diagram: You ‚Üí Claude ‚Üí Code ‚Üí Feedback ‚Üí Improved Code
- Show a real example of iteration (maybe from building 02Ship)
- On-screen text: "Collaboration > Magic Button"

## Production Notes

### Tone & Style
- **Conversational and warm** (friend teaching a friend)
- **Encouraging, not condescending** (assume intelligence, just lack of technical knowledge)
- **Use analogies** (director/crew, chef/recipe, intern/boss)
- **Show, don't just tell** (use the 02Ship platform as proof)

### Common Beginner Concerns to Address
- "Will I need to learn programming?" ‚Üí No, but you'll understand it better over time
- "What if I make mistakes?" ‚Üí Iteration is the whole point
- "Is this too technical for me?" ‚Üí If you can have a conversation, you can use Claude

### Things to Avoid
- Technical jargon without explanation
- Overwhelming with details (save for later videos)
- Making it sound too easy (set realistic expectations)
- Underselling what's possible (they CAN build real things)

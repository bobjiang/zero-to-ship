Section 2 - 3:30
Step 1

Push to GitHub

GitHub is where your code lives online. Vercel reads from it to deploy your site. Three commands is all it takes.

$ git add .
Staging all files...

$ git commit -m "Ready to deploy"
[main a3f2d1c] Ready to deploy
12 files changed, 847 insertions(+)

$ git push
✓ Everything up-to-date
Section 2 - 4:00
Step 2

Connect to Vercel

Sign up at vercel.com. Click "New Project." Import from GitHub. Vercel auto-detects Next.js. Click "Deploy."

That's it. Vercel handles everything else.

Section 6 - 10:00

After You Ship

Respond to Feedback

Read every piece. Thank everyone. If 3 people say the same thing, prioritize that fix. Use Claude to implement improvements.

Monitor Metrics

Check analytics daily for the first week. Look at visitors, sources, and clicks. Don't obsess -- look for patterns.

Lesson 7 Homework

Exercise 1: Plan Your MVP

Complete the MVP Planning Template: one-sentence pitch, problem/solution, 3 features max, tech stack, timeline. ~60 min

Exercise 2: Build Your MVP Core Features

Build your planned MVP from scratch using Claude. All 3 features working, committed to Git. ~3-7 days

Exercise 3: Deploy and Launch Your MVP

Deploy to Vercel, launch on 2+ platforms, get 5 users to try it, document feedback. ~2-3 hours

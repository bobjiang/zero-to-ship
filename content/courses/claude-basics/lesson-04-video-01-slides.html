Section 2 - 3:30
1
Code Artifact

Code Artifacts

The most common type. When you ask Claude to build something interactive, it generates working code. HTML you can open in your browser, or React components for larger projects.

Key point: Code artifacts are functional. They actually work.

/* Claude-generated HTML */ <!DOCTYPE html> <html> <head> <title>My Page</title> </head> <body> <h1>Hello World!</h1> <button onclick="alert('Hi!')"> Click Me </button> </body> </html>
Section 2 - 4:00
2
Document Artifact

Document Artifacts

Structured text created in Markdown format. Perfect for READMEs, project plans, guides, and any formatted text content.

Key point: Markdown is simple formatting -- no coding required to read it.

# Project README
## Getting Started
  • Install dependencies
  • Run the dev server
  • Open in browser
## Features
  • Responsive design
  • Dark mode support

Diagram Artifacts

Visual representations like flowcharts, process diagrams, and architecture overviews. Created as SVG images or Mermaid text diagrams.

Key point: Great for planning and visualizing your project structure.

Working with Artifacts

📋

Copy

Puts content on your clipboard. Paste into a text editor or code file.

📥

Download

Saves as a file directly to your computer. Great for images and diagrams.

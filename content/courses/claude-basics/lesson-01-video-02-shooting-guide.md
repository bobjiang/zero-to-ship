## Section 3: Safety Basics (5:00 - 7:30)

### The Golden Rule

**SCRIPT:**
> "Alright, the most important part of this video: safety. When you're working with AI-generated code, there's one golden rule I want you to remember: always read what Claude generates before you run it."

**Keypoints:**
- Always read code before running it
- You don't need to understand every line
- Look for things that seem off: file deletions, network requests to unknown places, requests for passwords
- When in doubt, ask Claude to explain what the code does

**Visual suggestions:**
- On-screen text (large): "Read before you run"
- Show an example of reviewing code briefly

### API Keys and Sensitive Information

**SCRIPT:**
> "Second safety rule: never share your API keys or passwords with anyone, and never paste them into public places. An API key is like a password for a service‚Äîif someone gets it, they can use your account and potentially charge you money."

**Keypoints:**
- API keys = passwords for services
- Never share them publicly (GitHub, forums, screenshots)
- Never paste them into Claude conversations (Claude doesn't need your passwords)
- If you accidentally share one, revoke it immediately and create a new one

**What to emphasize:**
> "If Claude ever asks you for a password or API key, something is wrong. Claude doesn't need your passwords to help you."

**Visual suggestions:**
- Show an example of what an API key looks like (redacted)
- On-screen text: "Never share API keys"
- Show where to find API key settings (e.g., in a service dashboard)

### Basic Code Review Checklist

**SCRIPT:**
> "I know you might be thinking, 'How can I review code if I don't know how to code?' Fair question. Here's a simple checklist for beginners:"

**Keypoints ‚Äî checklist to show on screen:**
1. Does Claude's explanation match what you asked for?
2. Are there any file operations you didn't request? (deleting files, accessing system folders)
3. Does the code reference external websites or services you didn't mention?
4. When in doubt, ask Claude: "Can you explain what this code does step by step?"

**SCRIPT (continued):**
> "You don't need to be a security expert. Just use common sense. If you asked for a calculator app and Claude's code is trying to access your email, that's a red flag. In practice, this almost never happens‚ÄîClaude is designed to be helpful and safe‚Äîbut it's a good habit to build from day one."

**Visual suggestions:**
- Show the checklist as on-screen graphics
- Keep it simple and non-intimidating
- On-screen text: "When in doubt, ask Claude to explain"

## Production Notes

### Tone & Style
- **Practical and reassuring** (this is easy, you can do this)
- **Step-by-step** (show every click, don't skip steps)
- **Non-intimidating** (safety section should inform, not scare)
- **Encouraging** (they're almost ready to start building!)

### Common Beginner Concerns to Address
- "Do I need to pay?" ‚Üí No, free plan works fine for now
- "I'm not technical enough to review code" ‚Üí Use the checklist; ask Claude to explain
- "What if I mess up the folder structure?" ‚Üí It's just folders; you can reorganize anytime
- "Is AI-generated code dangerous?" ‚Üí Generally safe, but good habits matter

### Things to Avoid
- Making setup sound complicated (it's three steps)
- Dwelling too long on safety (inform but don't frighten)
- Skipping steps in the screen recording (beginners need to see every click)
- Assuming familiarity with terms like "API key" without explaining
